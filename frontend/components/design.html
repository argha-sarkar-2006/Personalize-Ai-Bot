01

Design Philosophy

Aesthetic Direction
Space-grade Dark UI
Evokes mission-control terminals and financial data terminals — authoritative yet modern.
Information Density
High — Always Readable
Every pixel carries data. No decorative filler. Whitespace is structural, not aesthetic padding.
Motion Principle
Enter Once · Linger Never
Animations fire on viewport entry via IntersectionObserver, then stop. No looping distractions except intentional status pulses.
Depth System
3-Layer Stacking
Fixed atmospheric blobs → grid noise overlay → scrollable content. Creates illusion of depth without performance cost.
Color Role
Semantic Accent Model
Each hue carries a fixed meaning: Blue = primary data, Green = positive/profit, Red = cost/risk, Amber = warning/projection, Purple = secondary.
Font Pairing
Display + Terminal
Syne (geometric, heavy) for headings and labels. JetBrains Mono for all numbers, codes, and data — maintains monospace alignment in tables.

04

Layout Anatomy

The layout uses a single centered wrapper (max-width 1400px, 32px side padding) with a strict section-then-grid rhythm. Every section begins with a label row, followed by one of three grid systems depending on content complexity.
Full Width
Single-column card spanning the full content width. Used for primary trend charts (5-year revenue) and complex tables where horizontal scan matters.
.chart-full → width: 100%
2 : 1 Split
Primary chart on the left (2fr) paired with a donut / stat card on the right (1fr). Used for Quarterly + Segment Revenue section.
.charts-row → grid: 2fr 1fr; gap: 20px
50 / 50 Split
Equal-weight columns. Used for P&L Waterfall + Geography, Risk chart + Table, Milestones + Radar chart.
.chart-half → grid: 1fr 1fr; gap: 20px
KPI Auto-Grid
8 KPI cards in a responsive auto-fit grid. Minimum card width 190px — naturally flows from 4 columns to 2 on smaller screens.
.kpi-grid → repeat(auto-fit, minmax(190px,1fr))
05

Component Patterns

Component 01
KPI Card
Shows a single metric with animated counter, YoY change chip, icon, and colour-coded top border + glow via --kpi-color CSS variable.
--kpi-color: #3b9eff; ::before → 1px top gradient ::after → corner radial glow data-val → animated counter target hover: translateY(-3px) + shadow
Component 02
Chart Card (.c-card)
Universal chart container. Starts invisible (opacity 0, translateY 16px), reveals on IntersectionObserver with .visible class. Hover tints border to --border2.
opacity: 0 → 1 (on .visible) transform: translateY(16px) → 0 transition: .5s ease transitionDelay: i%4 × 70ms
Component 03
Animated Bar
Horizontal progress bar with data-pct attribute. Width animates from 0% to target on IntersectionObserver. Staggered with i × 100–120ms delays.
.bar-fill { width: 0 } IO fires → width: data-pct + '%' transition: 1.2s cubic-bezier (.16,1,.3,1) ← spring ease
Component 04
Animated Counter
JS animateValue() counts from 0 to target using requestAnimationFrame with ease-out quartic. Fires per KPI card via dedicated IntersectionObserver at 30% threshold.
ease = 1 - (1-t)^4 current = target × ease duration: 1400ms (KPI) / 2000ms (header)
Component 05
Risk Chip
Inline status pill with three states: HIGH (red tint), MED (amber tint), LOW (green tint). Built with a single class swap — no JS needed.
.risk-H → rgba(255,77,109,.15) .risk-M → rgba(255,184,48,.1) .risk-L → rgba(0,229,160,.1) font: JetBrains Mono 500 10px
Component 06
Section Title Divider
Flexbox row: uppercase 11px label on the left, auto-stretching 1px horizontal rule on the right via ::after pseudo-element. No extra markup.
.sec-title { display: flex } ::after { flex: 1; height: 1px; background: --border }
06

Background & Atmosphere Layers

Layer 1 · Base
--bg: #080b10
Near-black with a faint blue-grey tint. Not pure black — avoids harsh contrast; easier on eyes in dark mode.
Layer 2 · Grid Lines
body::after
Fixed 48×48px grid via repeating linear-gradient at 3% opacity. Gives the "data terminal" feel without heavy performance cost.
Layer 3 · Noise Texture
body::before (SVG feTurbulence)
Inline SVG fractalNoise filter at 0.03 opacity. Eliminates the "flat digital" look, mimics analogue screen grain.
Layer 4 · Glow Blobs
.blob1 / .blob2
Two radial blobs (500px, 400px) with 120px Gaussian blur. Top-left blue, bottom-right cyan. Animate opacity 0.5↔1 over 6–7s. Position: fixed so they don't scroll.
Layer 5 · Cards
--card: #111820
Raised 3px above background. Cards use 1px solid border at 6% white opacity. Hover state shifts border to 12% blue-tinted rgba.
Layer 6 · Content
z-index: 1
All scrollable content in .wrapper sits at z-index:1 above the fixed atmosphere layers. Chart tooltips and active states render above via Chart.js internal z management.
07

Chart.js Configuration Patterns

Pattern 01
Global Defaults
Set once before any Chart() call. Overrides Chart.js defaults to match dark theme — avoids repeating tooltip/grid config in every chart.
Chart.defaults.color = '#4a5a70' Chart.defaults.font.family = "'Syne',sans-serif"
Pattern 02
baseOpts() Factory
A shared options object factory returning consistent tooltip, grid, tick styling, and animation config. Each chart spreads baseOpts() then overrides only what it needs.
function baseOpts(extra={}) { return { plugins: { tooltip: ... }, scales: { x:…, y:… }, animation: { duration:1400 }, ...extra } }
Pattern 03
Dual Y-Axis
Projection chart uses yAxisID: 'y' for ₹ values and yAxisID: 'y2' (right-positioned) for EBITDA %. Right axis ticks coloured amber to match the dataset line.
y2: { position: 'right', grid: { display: false }, ticks: { color: AMBER }, min: 20, max: 30 }

10

Data-Driven Rendering Patterns

data-val / data-prefix / data-suffix
HTML data-attributes on .kpi
KPI counter target, currency prefix (₹), and unit suffix (Cr, %, pts) are stored as HTML data attributes — JS reads them without touching source data arrays.
innerHTML injection loops
forEach → innerHTML +=
Geography bars, HR bars, D&I cards, vertical table rows, segment legend, and timeline items are all built from JS arrays with template literal injection. Zero DOM libraries needed.
--kpi-color CSS variable
Per-card colour theming
Each KPI card sets --kpi-color inline. The ::before and ::after pseudo-elements consume it, so border, top line, and corner glow all update with one variable.
data-pct on .bar-fill
Bar width from attribute
All progress bars start at width:0 via CSS. IntersectionObserver reads data-pct and sets width to trigger the transition. Staggered via setTimeout(i × delay).
Semantic colour logic
attrition > 15 → RED else GREEN
HR bar attrition badge colour is conditionally assigned: red if >15%, green otherwise. All sentiment-coded displays use inline ternary expressions, not CSS classes.
Shared IntersectionObserver
barObs reused across sections
A single barObs instance is registered for geography, vertical table, HR, and D&I sections. Avoids creating separate observers for each section — one IO handles all bar reveals.
11

Responsive Strategy

Breakpoint
@media max-width: 900px
Single breakpoint only. Below 900px, all multi-column grids (.charts-row, .chart-thirds, .chart-half) collapse to single column. KPI grid is auto-fit — wraps naturally.
Header
flex-direction: column
Header flips from row (title left, revenue right) to column stacked. Revenue block switches text-align from right to left. Gap of 16px between sections.
Typography
clamp() for fluid type
Main h1 uses clamp(28px, 4vw, 48px) — scales from 28px on mobile to 48px on wide screen. KPI values use fixed 28px (already compact).
Charts
responsive: true (Chart.js default)
All Chart.js canvases have responsive:true so they resize with their container. The donut chart uses fixed width (160px canvas) — wrapped in flexbox that reflows.
